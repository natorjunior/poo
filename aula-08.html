Listas e Coleções

Armazenar e processar múltiplos dados de forma eficiente

ArrayList, LinkedList, HashSet, HashMap

O problema: Arrays fixos

Quando você precisa de muitos dados, arrays não crescem.

String[] nomes = new String[3]; // Fixo: apenas 3 elementos
nomes[0] = "Ana";
nomes[1] = "Bruno";
nomes[2] = "Carla";

// nomes[3] = "Diana"; // ❌ ArrayIndexOutOfBoundsException!
O array não pode crescer. Você precisa criar um novo array, maior.

A solução: Coleções dinâmicas

Coleções crescem automaticamente conforme você adiciona elementos.

import java.util.*;

List<String> nomes = new ArrayList<>(); // Cresce automaticamente
nomes.add("Ana");
nomes.add("Bruno");
nomes.add("Carla");
nomes.add("Diana");

System.out.println(nomes.size()); // 4 ✅
Sem especificar tamanho. Cresce conforme necessário.

Por que coleções?

Crescimento dinâmico
Adicione elementos sem se preocupar com limite.
Operações eficientes
Busca, inserção, remoção otimizadas por tipo.
Interface consistente
List, Set, Map — padrões previsíveis.

ArrayList: Começando simples

Criar uma lista vazia e adicionar elementos.

import java.util.ArrayList;
import java.util.List;

List<String> frutas = new ArrayList<>();

frutas.add("Maçã");
frutas.add("Banana");
frutas.add("Laranja");

System.out.println(frutas);
// Saída: [Maçã, Banana, Laranja]
Tipo genérico <String> define o que a lista armazena.

ArrayList: Acessando elementos

Recupere elementos pela posição ou tamanho.

List<String> frutas = new ArrayList<>();
frutas.add("Maçã");
frutas.add("Banana");
frutas.add("Laranja");

System.out.println(frutas.get(0));  // Maçã
System.out.println(frutas.get(1));  // Banana
System.out.println(frutas.size());  // 3
System.out.println(frutas.contains("Banana")); // true
Posição começa em 0. Acesso é muito rápido O(1).

ArrayList: Iteração

Percorrer todos os elementos com for-each.

List<String> frutas = new ArrayList<>();
frutas.add("Maçã");
frutas.add("Banana");
frutas.add("Laranja");

// Modo 1: for-each
for (String fruta : frutas) {
  System.out.println(fruta);
}

// Modo 2: for tradicional
for (int i = 0; i < frutas.size(); i++) {
  System.out.println(frutas.get(i));
}
For-each é mais legível. For tradicional permite acessar índice.

ArrayList: Remover e substituir

Modifique o conteúdo da lista.

List<String> frutas = new ArrayList<>();
frutas.add("Maçã");
frutas.add("Banana");
frutas.add("Laranja");

frutas.set(1, "Melancia"); // Substitui Banana por Melancia
System.out.println(frutas);
// [Maçã, Melancia, Laranja]

frutas.remove(0); // Remove Maçã (índice 0)
System.out.println(frutas);
// [Melancia, Laranja]
set() substitui, remove() deleta.

LinkedList: Inserção no meio

Insira elementos em qualquer posição, não apenas no final.

import java.util.LinkedList;
import java.util.List;

List<Integer> numeros = new LinkedList<>();
numeros.add(10);
numeros.add(20);
numeros.add(30);

System.out.println(numeros);
// [10, 20, 30]

numeros.add(1, 15); // Insere 15 na posição 1
System.out.println(numeros);
// [10, 15, 20, 30]
Inserção no meio é muito mais rápida em LinkedList que em ArrayList.

LinkedList: Remoção por valor

Remova removendo pelo valor, não pela posição.

List<Integer> numeros = new LinkedList<>();
numeros.add(10);
numeros.add(15);
numeros.add(20);
numeros.add(30);

System.out.println(numeros);
// [10, 15, 20, 30]

numeros.remove(Integer.valueOf(15)); // Remove o valor 15
System.out.println(numeros);
// [10, 20, 30]

boolean removeu = numeros.remove(Integer.valueOf(100)); // Valor não existe
System.out.println(removeu); // false
remove(value) é diferente de remove(index).

HashSet: O problema de duplicatas

Você quer garantir que não haja valores repetidos.

import java.util.*;

List<String> linguagens = new ArrayList<>();
linguagens.add("Java");
linguagens.add("Python");
linguagens.add("Java");  // Duplicata!
linguagens.add("Go");

System.out.println(linguagens.size()); // 4
System.out.println(linguagens);
// [Java, Python, Java, Go]  ← Tem Java duas vezes
ArrayList permite duplicatas. Você precisa verificar manualmente ou usar Set.

HashSet: Automático, sem duplicatas

Adicione valores; duplicatas são ignoradas.

import java.util.HashSet;
import java.util.Set;

Set<String> linguagens = new HashSet<>();
linguagens.add("Java");
linguagens.add("Python");
linguagens.add("Java");  // Ignorado, pois já existe
linguagens.add("Go");

System.out.println(linguagens.size()); // 3
System.out.println(linguagens);
// [Python, Go, Java]  ← Sem duplicatas, ordem arbitrária
HashSet automaticamente descarta duplicatas. Ordem não é garantida.

HashSet: Busca de pertencimento

Verifique se um elemento está no conjunto.

Set<String> linguagens = new HashSet<>();
linguagens.add("Java");
linguagens.add("Python");
linguagens.add("Go");

// Verificar se existe
System.out.println(linguagens.contains("Java"));   // true
System.out.println(linguagens.contains("C++"));    // false

// Remover um elemento
linguagens.remove("Python");
System.out.println(linguagens.size()); // 2
System.out.println(linguagens.contains("Python")); // false
contains() é muito rápido em HashSet (O(1)).

HashMap: Mapeando chave-valor

Armazene dados associados: chave → valor

import java.util.HashMap;
import java.util.Map;

Map<String, Integer> notas = new HashMap<>();

notas.put("Ana", 95);
notas.put("Bruno", 87);
notas.put("Carla", 92);

System.out.println(notas);
// {Ana=95, Bruno=87, Carla=92}

System.out.println(notas.get("Ana")); // 95
System.out.println(notas.get("Diana")); // null (não existe)
Chaves devem ser únicas. Valores podem repetir.

HashMap: Operações comuns

Adicione, busque, remova e verifique.

Map<String, Integer> notas = new HashMap<>();
notas.put("Ana", 95);
notas.put("Bruno", 87);
notas.put("Carla", 92);

// Verificar se chave existe
System.out.println(notas.containsKey("Ana"));    // true
System.out.println(notas.containsKey("Diana"));  // false

// Verificar se valor existe
System.out.println(notas.containsValue(95));     // true
System.out.println(notas.containsValue(100));    // false

// Tamanho do mapa
System.out.println(notas.size()); // 3
containsKey() é muito mais rápido que containsValue().

HashMap: Atualizando valores

Modifique valores e remova pares.

Map<String, Integer> notas = new HashMap<>();
notas.put("Ana", 95);
notas.put("Bruno", 87);
notas.put("Carla", 92);

// Atualizar uma nota
notas.put("Ana", 98); // Substitui o 95 por 98
System.out.println(notas.get("Ana")); // 98

// Remover um par
notas.remove("Bruno");
System.out.println(notas.size()); // 2

// Obter com padrão se não existe
Integer notaDiana = notas.getOrDefault("Diana", 0);
System.out.println(notaDiana); // 0
getOrDefault() evita NullPointerException.

HashMap: Iteração

Percorra chaves, valores ou pares.

Map<String, Integer> notas = new HashMap<>();
notas.put("Ana", 95);
notas.put("Bruno", 87);
notas.put("Carla", 92);

// Iterar pares (chave-valor)
for (String nome : notas.keySet()) {
  System.out.println(nome + ": " + notas.get(nome));
}

// Ou usando entrySet (mais eficiente)
for (Map.Entry<String, Integer> entry : notas.entrySet()) {
  System.out.println(entry.getKey() + " → " + entry.getValue());
}
entrySet() é mais rápido que keySet().

Comparação: qual coleção usar?

ArrayList
Acesso por índice rápido. Inserir/remover no meio → lento.
LinkedList
Inserir/remover rápido. Acesso por índice → lento.
HashSet
Sem duplicatas. Sem ordem. Busca muito rápida.
HashMap
Chave-valor. Sem ordem. Busca muito rápida.

Cenário 1: Lista de alunos

Use ArrayList quando você acessa por índice frequentemente.

List<String> alunos = new ArrayList<>();
alunos.add("Ana");
alunos.add("Bruno");
alunos.add("Carla");

// Ler de cima para baixo
for (int i = 0; i < alunos.size(); i++) {
  System.out.println((i + 1) + ". " + alunos.get(i));
}

// Saída:
// 1. Ana
// 2. Bruno
// 3. Carla
ArrayList é ideal para listas ordenadas com acesso por posição.

Cenário 2: Remover em lote

Use LinkedList quando precisa inserir/remover frequentemente.

List<Integer> fila = new LinkedList<>();
fila.add(1);
fila.add(2);
fila.add(3);
fila.add(4);

// Operações de fila: remover do início, adicionar no fim
fila.remove(0);      // Remove primeiro
fila.add(5);         // Adiciona último

System.out.println(fila); // [2, 3, 4, 5]
LinkedList é ideal para filas e operações de pilha.

Cenário 3: Garantir unicidade

Use HashSet quando não pode ter duplicatas.

Set<String> emails = new HashSet<>();
emails.add("user@example.com");
emails.add("admin@example.com");
emails.add("user@example.com");  // Ignorado

System.out.println(emails.size()); // 2

// Verificar rapidamente
if (emails.contains("user@example.com")) {
  System.out.println("Email já registrado");
}
HashSet é ideal para validação de unicidade e rápida busca.

Cenário 4: Associação rápida

Use HashMap quando precisa de chave → valor.

Map<String, String> config = new HashMap<>();
config.put("host", "localhost");
config.put("port", "8080");
config.put("db", "myapp");

// Lookup rápido
String host = config.get("host"); // "localhost"

// Inicializar com padrão
String timeout = config.getOrDefault("timeout", "30");
System.out.println(timeout); // "30"
HashMap é ideal para dicionários, configurações e caches.

Threads e Concorrência

Executar múltiplas tarefas simultaneamente no mesmo programa

Sincronização, race conditions e estados de thread

O que é uma thread?

Uma thread é um fluxo de execução independente que permite paralelizar tarefas dentro do mesmo processo.

Problema: Execução sequencial

Sem threads, tarefas executam uma após a outra (lento).
Linha do tempoThread principal
Ler arquivo (2s)
Processar (1s)

Problema: Execução sequencial

Sem threads, tarefas executam uma após a outra (lento).
Linha do tempoThread principal
Ler arquivo (2s)
Processar (1s)

A thread principal precisa terminar a tarefa 1 para começar a tarefa 2. Tempo total = 2s + 1s = 3s.


public class Tarefa1 {
  public static void main(String[] args) {
    System.out.println("Iniciando...");
    
    // Tarefa 1: lê arquivo (2 segundos)
    System.out.println("Lendo arquivo...");
    Thread.sleep(2000);
    
    // Tarefa 2: processa dados (1 segundo)
    System.out.println("Processando...");
    Thread.sleep(1000);
    
    System.out.println("Concluído");
    // Total: 3 segundos (espera uma terminar para iniciar outra)
  }
}
Tempo total: 3 segundos (sequencial)

Duas threads em paralelo

Tarefas rodam simultaneamente (comparar com sequencial).

public class Main {
  public static void main(String[] args) {
    // Criar 2 threads
    Thread t1 = new Thread(() -> {
      System.out.println("Thread-1 lendo (2s)...");
      Thread.sleep(2000);
      System.out.println("Thread-1 concluída");
    });

    Thread t2 = new Thread(() -> {
      System.out.println("Thread-2 processando (1s)...");
      Thread.sleep(1000);
      System.out.println("Thread-2 concluída");
    });

    t1.start();
    t2.start();
    System.out.println("Main continua...");
  }
}
Total: ~2 segundos (paralelo) vs 3 segundos (sequencial)

public class Main {
  public static void main(String[] args) throws InterruptedException {
    Thread t1 = new Thread(() -> {
      System.out.println("Tarefa 1 iniciada");
      Thread.sleep(2000);
      System.out.println("Tarefa 1 concluída");
    });

    Thread t2 = new Thread(() -> {
      System.out.println("Tarefa 2 iniciada");
      Thread.sleep(1000);
      System.out.println("Tarefa 2 concluída");
    });

    t1.start();
    t2.start();

    // Aguardar ambas terminarem
    t1.join(); // Pausa aqui até t1 terminar
    t2.join(); // Pausa aqui até t2 terminar

    System.out.println("Todas as tarefas concluídas!");
  }
}
join() pausa thread main até que a filha termine.

Estados de uma thread

NEW
Thread criada, ainda não iniciou.
→
RUNNABLE
Pronta ou executando CPU.
⇄
BLOCKED / WAITING
Aguardando lock ou evento.
→
TERMINATED
run() terminou.
Threads alternam entre RUNNABLE e BLOCKED várias vezes até encerrar.
